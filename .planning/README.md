# Planning Files

This directory holds the living project metadata for the canonical library. The files here are small on purpose: they capture the current project definition, roadmap, and state without duplicating the source bundles themselves.

## Files

| File | Purpose |
|---|---|
| `PROJECT.md` | Project definition, core value, active requirements, constraints, and key decisions. |
| `ROADMAP.md` | Phase structure and success criteria for the current milestone. |
| `STATE.md` | Short-term project memory: current position, recent decisions, blockers, and session continuity. |
| `quick/` | Scratch space for quick-task runs and their working notes. |

## How To Read It

Start with `PROJECT.md` for the stable project definition, then read `ROADMAP.md` for the current work structure, and finally `STATE.md` for where execution last stopped.

## Maintenance Rules

- Keep the files concise and current; they should describe the repository as it exists now.
- Update `PROJECT.md` when the canonical scope or constraints change.
- Update `ROADMAP.md` when phase structure changes.
- Update `STATE.md` after significant activity so quick resumption stays accurate.
