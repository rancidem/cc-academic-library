# cc-academic-library/site

Static HTML reference console for the cc-academic-library. ~216 pages. No build tool, no framework, no npm — plain HTML + one CSS file + one JS file. GitHub Pages compatible.

> **Location:** This directory lives inside the `cc-academic-library` repo as `site/`. The source library (skills, commands, agents) is the parent directory (`../`).

## At a Glance

| Area | Path | Description |
|---|---|---|
| Home | `index.html` | Site root and section index |
| Skills | `skills/` | 142 skill entries in 13 domain directories |
| Commands | `commands/` | 37 slash command entries |
| Agents | `agents/` | 11 subagent entries |
| Tools | `tools/` | Auxiliary tools (WCN) |
| Templates | `templates/` | Prompt and document templates |

| Generator | `generate.js` | Regenerates skill/command/agent/tool pages from source library |

## Regenerating Pages

Skill, command, agent, and tool entry pages are generated from the source library. To regenerate:

```bash
# Prerequisites: Node.js (no npm required)
# Run from inside cc-academic-library/site/
node generate.js

# Or from the cc-academic-library root:
node site/generate.js
```

`generate.js` reads the parent directory (`../`) as source and writes all entry pages plus a fresh `search-index.js` into `site/`. Paths are `__dirname`-relative — no manual adjustment needed as long as the file stays at `site/generate.js`.

Each generated skill/command/agent page now includes:
- **RAW FILES** — download links to the original source `.md`, reference files, scripts, and assets (copied into `raw/` subdirectories)
- **FULL CONTENT** — the complete source file body rendered in a scrollable block

**What the generator produces:**

| Target | Source | Count |
|---|---|---|
| `skills/{domain}/{slug}/index.html` | `cc-academic-library/skills/` | 142 |
| `commands/{slug}/index.html` | `cc-academic-library/commands/` | 37 |
| `agents/{slug}/index.html` | `cc-academic-library/agents/` | 11 |
| `tools/wcn/index.html` | Hard-coded in `generate.js` | 1 |
| `search-index.js` | All of the above | 191 entries |

Section index pages (`skills/index.html`, `commands/index.html`, etc.) and domain index pages (`skills/{domain}/index.html`) are hand-authored and not regenerated.

## Deploying to GitHub Pages

1. Push this directory to a GitHub repository
2. Go to **Settings → Pages**
3. Set source to **Deploy from a branch**, branch `main`, folder `/` (root)
4. The site will be live at `https://{username}.github.io/{repo-name}/`

No `_config.yml` required. For a custom domain, add a `CNAME` file to the root containing your domain.
