# Start Here

Use this page if you are new to Claude Code-style skills, commands, or plugin repos and want the shortest path to something usable.

## What This Library Is

- This project is a personal index of five cloned repositories.
- Each repo contains reusable commands, agents, skills, tools, or supporting docs.
- `LIBRARY.md` is the main dashboard.
- `library/repositories.md` is the main comparison page.

## Beginner Path

1. Open `LIBRARY.md`.
2. Pick one repo from the repository table.
3. Open that repo's section in `library/repositories.md`.
4. Follow the `What do I do first?` line, then the `Install / setup` line, then the `How to use` line.

## Recommended First Choices

- Easiest repo: [MySkills](repositories.md#myskills)
- Easiest skill: `paper-visualizer` in [MySkills](repositories.md#myskills)
- Easiest install: [scientific-agent-skills](repositories.md#scientific-agent-skills)
- Broadest catalog: [scientific-agent-skills](repositories.md#scientific-agent-skills)
- Best full writing workflow: [wtf-p](repositories.md#wtf-p)
- Best scientific writing workflow: [claude-scientific-writer](repositories.md#claude-scientific-writer)

## When To Choose Each Repo

- [MySkills](repositories.md#myskills): choose this when you want the smallest example and the fastest way to understand how one skill bundle works.
- [academic-paper-skills](repositories.md#academic-paper-skills): choose this when you want a short paper-planning and paper-writing workflow without much setup.
- [wtf-p](repositories.md#wtf-p): choose this when you want an installable writing system with clear commands and stronger workflow structure.
- [claude-scientific-writer](repositories.md#claude-scientific-writer): choose this when you want research-backed scientific writing plus plugin, CLI, and package install options.
- [scientific-agent-skills](repositories.md#scientific-agent-skills): choose this when you want the biggest scientific skill catalog and are willing to scan more material.

## Direct File Routes

- Want the raw skill files: open [library/skills.md](skills.md)
- Want tool roots or package files: open [library/tools.md](tools.md)
- Want supporting docs and install references: open [library/resources.md](resources.md)
- Want everything in one table: open [library/inventory.md](inventory.md)

## Read The Repo Summaries This Way

- `What it is`: plain description of the repo.
- `What do I do first?`: first file or folder to open if you are not sure where to start.
- `Install / setup`: the shortest path to get it working locally.
- `Download / raw`: how to get the files directly.
- `How to use`: the practical use case.
- `Why choose it`: why this repo is the right choice instead of another one.
- `Caution`: what makes it heavier, narrower, or harder to use.
