# cc-academic Library

This folder is the working index for the project, but `../LIBRARY.md` is the dashboard you should open first.

## What Lives Here

- [`../LIBRARY.md`](../LIBRARY.md) - master catalog and personal dashboard
- [Library inventory](inventory.md) - full command, agent, skill, and tool catalog
- [Commands](commands.md) - command entry index with summaries and raw links
- [Agents](agents.md) - agent entry index with summaries and raw links
- [Repository summaries](repositories.md) - detailed per-repo comparison and entry points
- [Skills inventory](skills.md) - indexed skill-file catalog with summaries, usage notes, open links, and raw links
- [Tools](tools.md) - reusable tool roots, packages, and helper utilities
- [Resources](resources.md) - supporting references, guides, and maintenance material
- [Entry template](entry-template.md) - shared schema for commands, agents, skills, tools, and resources
- [Tag taxonomy](taxonomy.md) - controlled vocabulary for search and filtering

## How To Use It

1. Start with `../LIBRARY.md` for the current state and best entry points.
2. Use `inventory.md` when you want the complete catalog across commands, agents, skills, and tools.
3. Use `repositories.md` when you want richer per-repo summaries.
4. Use `commands.md`, `agents.md`, `skills.md`, and `tools.md` when you want a type-specific index.
5. Use `resources.md` when you want reusable references, not just repo navigation.
6. Tag new entries with the controlled taxonomy before expanding the index.

## Status

The library is organized for fast personal use and ongoing maintenance.
